파이어베이스 콘솔에서 **Authentication(인증)**을 활성화하는 과정을 아주 상세하게, 클릭할 위치까지 짚어서 설명해 드릴게요. 이 작업이 완료되어야 우리 프로그램이 "진짜 열쇠"를 가질 수 있습니다.

1단계: Authentication 메뉴 진입
파이어베이스 콘솔에 접속하여 본인의 프로젝트(cmlogin-9d49d)를 클릭합니다.

왼쪽 메뉴 바에서 빌드(Build) 항목을 클릭한 뒤, Authentication을 선택합니다.

중앙에 보이는 시작하기(Get Started) 버튼을 클릭합니다.

2단계: 로그인 방법(이메일/비밀번호) 설정
인증 서버가 켜졌다면, 이제 어떤 방식으로 로그인할지 정해야 합니다.

상단 탭에서 **Sign-in method(로그인 방법)**를 클릭합니다.

새 제공업체 추가(Add new provider) 버튼을 누릅니다.

목록에서 **이메일/비밀번호(Email/Password)**를 선택합니다.

첫 번째 항목인 설정(Enable) 토글을 '사용 설정됨'으로 바꿉니다. (두 번째 '이메일 링크'는 안 켜도 됩니다.)

하단의 저장(Save) 버튼을 누릅니다.

3단계: 관리자 계정 생성 (실제 사용할 아이디)
이제 관리자님이 직접 사용할 아이디를 수동으로 하나 만들어야 합니다.

상단 탭에서 **Users(사용자)**를 클릭합니다.

오른쪽의 사용자 추가(Add user) 버튼을 누릅니다.

이메일 주소와 비밀번호를 입력합니다. (예: admin@test.com / password123)

주의: 여기서 만든 이메일과 비번이 나중에 앱 로그인 창에 입력할 정보가 됩니다.

사용자 추가 버튼을 눌러 완료합니다.

4단계: 고유 식별자(UID) 확인하기
사용자가 생성되면 목록에 User UID라는 긴 문자열이 생깁니다.

UID 예시: A1b2C3d4E5f6G7...

이 UID는 전 세계에서 유일한 값입니다. 나중에 보안 규칙에서 **"이 UID를 가진 사람만 관리자다"**라고 지정할 때 사용합니다. 이 값을 따로 복사해두시면 좋습니다.

5단계: 이제 코드가 작동하는 원리
이렇게 설정을 마치면, 이제 login.js에서 작성할 signInWithEmailAndPassword 함수가 파이어베이스 서버에 접속해 다음과 같이 확인합니다.

프로그램: "서버야, 이 이메일이랑 비번 맞니?" 파이어베이스: "응, 내가 방금 콘솔에서 만든 그 사람 맞네! 로그인을 허가하고 인증 토큰을 보내줄게."

이 인증 토큰이 생성되어야만 우리가 앞서 설정한 **보안 규칙(Rules)**의 auth != null 문을 통과할 수 있게 됩니다.


데이터를 입력하는 화면이 익숙하지 않으시면 충분히 헷갈릴 수 있습니다! 마우스를 올려야 하는 정확한 위치와 방법을 다시 한번 짚어 드릴게요.

💡 정확한 위치 찾기
파이어베이스 콘솔의 Realtime Database 화면 중앙을 보시면 데이터 구조가 보입니다.

가장 꼭대기 줄: 프로젝트 이름(예: cmlogin-9d49d-default-rtdb)이 적힌 가장 첫 번째 줄에 마우스를 가져다 대세요.

숨겨진 버튼: 평소에는 안 보이다가, 마우스를 그 줄 근처로 가져가면 오른쪽에 파란색 + (더하기) 모양 버튼이 나타납니다.

📂 단계별로 방 만들기 (따라하기)
마우스를 올려서 버튼을 찾으셨다면 아래 순서대로 클릭해 보세요.

1. admins 라는 큰 방 만들기
최상단 줄의 + 를 누릅니다.

이름(Name) 칸에 admins라고 씁니다. (값 칸은 비워두세요)

**추가(Add)**를 누르거나 엔터를 쳐서 저장합니다.

2. 내 UID(또는 부관리자 UID) 방 만들기
방금 만든 admins 글자 줄에 마우스를 올리면, 또 그 줄 오른쪽에 + 버튼이 생깁니다. 그걸 누르세요.

이름(Name) 칸에 아까 복사한 긴 UID 번호를 붙여넣습니다.

이번에도 값은 비워둔 채로 그 줄 오른쪽에 새로 생긴 + 를 또 누릅니다. (자식 노드 생성)

3. 역할(role) 명찰 달아주기
UID 아래에 새로 생긴 칸에 다음과 같이 입력합니다.

이름(Name): role

값(Value): admin (또는 부관리자라면 sub_admin)

최종적으로 추가(Add) 버튼을 누르면 완성입니다!

✅ 완성된 모습 체크
입력이 잘 되었다면 아래와 같이 계단 모양으로 펼쳐져야 합니다.

admins

여기에_긴_UID_번호
